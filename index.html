2. Grundsatz: Keine Datenerhebung

Modbus++ erhebt, speichert oder überträgt keine personenbezogenen Daten an externe Server oder Dritte.

Die App kommuniziert ausschließlich mit Modbus-Geräten, die der Benutzer selbst konfiguriert. Alle gespeicherten Daten verbleiben ausschließlich lokal auf dem Gerät.

3. Lokal gespeicherte Daten

Die App speichert folgende Daten ausschließlich lokal auf dem Gerät:

  • Gerätekonfigurationen: Name, IP-Adresse, Port, Unit-ID und Registeradressen der konfigurierten Modbus-Geräte
  • Messwert-Aufzeichnungen: Zeitgestempelte Messwerte aus Polling-Sitzungen (in lokaler SQLite-Datenbank)
  • Alarm-Ereignisse: Protokoll ausgelöster Alarme mit Zeitstempel und Wert
  • App-Einstellungen: Farbschema, Sprache, Widget-Konfiguration (in SharedPreferences)

Diese Daten werden nicht verschlüsselt, da sie keine personenbezogenen Informationen enthalten. Sie können jederzeit in der App gelöscht werden.

4. Berechtigungen und deren Verwendung

INTERNET Netzwerkzugriff
Wird ausschließlich für TCP-Verbindungen zu Modbus-Geräten im lokalen Netzwerk verwendet, die der Benutzer selbst konfiguriert. Es werden keine Daten an das Internet gesendet.

BLUETOOTH_SCAN BLUETOOTH_CONNECT
Bluetooth-Kommunikation
Wird verwendet, um Bluetooth-Modbus-Geräte in der Nähe zu finden und eine serielle Verbindung (SPP-Profil) herzustellen. Gerätedaten werden nicht gespeichert oder übertragen.

NFC NFC-Tag-Kommunikation
Wird verwendet, um Verbindungsprofile (Name, IP, Port) auf NFC-Tags zu schreiben oder davon zu lesen. Es findet keine Kommunikation mit externen Diensten statt.

POST_NOTIFICATIONS Benachrichtigungen
Wird verwendet, um lokale Alarm-Benachrichtigungen anzuzeigen, wenn konfigurierte Schwellwerte überschritten werden oder ein Verbindungsverlust eintritt. Es werden keine Push-Dienste oder externe Server verwendet.

ACCESS_NETWORK_STATE Netzwerkstatus
Wird verwendet, um die Verfügbarkeit einer Netzwerkverbindung zu prüfen.

5. Dritte und Analyse-Tools

Die App verwendet keine Analyse-Dienste (Google Analytics, Firebase Analytics o. ä.), keine Werbung und keine Crash-Reporting-Dienste, die Daten an externe Server übertragen.

Folgende Open-Source-Bibliotheken werden eingebunden, die selbst keine Daten erfassen:

  • ZXing (QR-Code-Generierung und -Scan) – Apache 2.0
  • usb-serial-for-android (USB-Seriell-Kommunikation) – LGPL 2.1
  • Google Play Services Wearable API (Wear OS Datenübertragung, nur lokal zwischen Telefon und Uhr)

6. Wear OS

Wenn ein Wear-OS-Gerät gekoppelt ist, überträgt die App Messwerte über die lokale Wearable Data API zwischen Telefon und Uhr. Diese Übertragung findet ausschließlich lokal zwischen den gekoppelten Geräten statt und wird nicht über externe Server geleitet.

7. Ihre Rechte (DSGVO)

Da die App keine personenbezogenen Daten erhebt, entfallen die üblichen DSGVO-Auskunfts- und Löschpflichten gegenüber dem Anbieter. Alle lokal gespeicherten Daten können Sie jederzeit selbst in der App löschen:

  • Messwert-Aufzeichnungen: LogScreen → Sitzung löschen
  • Alarm-Ereignisse: AlarmScreen → Alle löschen
  • Gerätekonfigurationen: Strukturliste → Struktur löschen

2. Core Principle: No Data Collection

Modbus++ does not collect, store, or transmit any personal data to external servers or third parties.

The app communicates exclusively with Modbus devices that the user configures themselves. All stored data remains solely on the local device.

3. Locally Stored Data

The app stores the following data exclusively locally on the device:

  • Device configurations: Name, IP address, port, unit ID, and register addresses of configured Modbus devices
  • Measurement recordings: Timestamped measurement values from polling sessions (in a local SQLite database)
  • Alarm events: Log of triggered alarms with timestamp and value
  • App settings: Color scheme, language, widget configuration (in SharedPreferences)

This data is not encrypted as it does not contain personal information. It can be deleted at any time within the app.

4. Permissions and Their Use

INTERNET Network Access
Used exclusively for TCP connections to Modbus devices on the local network configured by the user. No data is sent to the internet.

BLUETOOTH_SCAN BLUETOOTH_CONNECT
Bluetooth Communication
Used to discover nearby Bluetooth Modbus devices and establish a serial connection (SPP profile). Device data is not stored or transmitted externally.

NFC NFC Tag Communication
Used to write or read connection profiles (name, IP, port) to/from NFC tags. No communication with external services takes place.

POST_NOTIFICATIONS Notifications
Used to display local alarm notifications when configured thresholds are exceeded or a connection loss occurs. No push services or external servers are used.

ACCESS_NETWORK_STATE Network State
Used to check the availability of a network connection.

5. Third Parties and Analytics

The app uses no analytics services (Google Analytics, Firebase Analytics, etc.), no advertising, and no crash reporting services that transmit data to external servers.

The following open-source libraries are included, none of which collect data:

  • ZXing (QR code generation and scanning) – Apache 2.0
  • usb-serial-for-android (USB serial communication) – LGPL 2.1
  • Google Play Services Wearable API (Wear OS data transfer, local only between phone and watch)

6. Wear OS

When a Wear OS device is paired, the app transmits measurement values via the local Wearable Data API between phone and watch. This transfer occurs exclusively locally between paired devices and is not routed through external servers.

7. Your Rights (GDPR)

Since the app does not collect personal data, standard GDPR disclosure and deletion obligations toward the provider do not apply. All locally stored data can be deleted at any time within the app:

  • Measurement recordings: LogScreen → Delete session
  • Alarm events: AlarmScreen → Delete all
  • Device configurations: Structure list → Delete structure
